CodeCook.io API

The application programming interface (API) exposes nearly all code on our website. This allows code from the site to be directly inserted into your work. Look at how to access the API or view the resources.

API accesss

The API is still under heavy development. You can access this it via the url "http://codecook.io/api/dev/". We will do our best to keep the interface stable, but things will change. If you use the API, please let us know and we will keep you posted on changes.

Authentication

Currently all access to the API requires an user account and API key. Your API key can be found on your user profile page.

When requesting a resource from the API you need to identify yourself by adding the following GET parameters to the request:

  • username=[username]
  • api_key=[api_key]

For example, a request including authentication for user "johndoe" with API key "g9sda17fgd35sdfg9823as5" is: 

http://codecook.io/api/dev/concept/1/?username="johndoe"&api_key="g9sda17fgd35sdfg9823as5"

Fair use

Feel free to use the API. You are free to do almost anything with it. We will however take action if either the requests rate is through the roof or when we feel the usage is not in line with our (very permissive) license.

Do try to keep the requests rate low. For example if you imlement a interactive search field that updates search results as you type, perform a lookup only after input has not changed for a small fraction of time (not every character change). This allows us to keep response times high for everyone.

Usage

A REST API is exposed through the domain codecook.io. Currently most aspects are available for reading through the API. Create and update actions are still in early stages of development. Valid request to the server returns a JSON file.

The API url is:

http://codecook.io/api/dev/

You can try querying a page in directly from the (linux) command line:

curl http://codecook.io/api/dev/?username="johndoe"&api_key="g9sda17fgd35sdfg9823as5"

Or use your browser and navigate to the page:

http://codecook.io/api/dev/?username="johndoe"&api_key="g9sda17fgd35sdfg9823as5"&format=json

For a list of available resource see resources.

Resources

All data is returned as JSON and look roughly the same. A list of resources will be returned as "objects" possibly with a "meta" section that contains pagination information. Single resources generally contain more data for that specific resource.

Here are some example endpoints with their corresponding replies:

  • Top level schema
  • Search concept
  • Concept list
  • Concept detail
  • Method detail
